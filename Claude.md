# Instrucciones de trabajo para Claude Code

## Contexto del proyecto
Este proyecto es un SaaS/micro-SaaS B2B en fase de validación.
Stack principal: Python y SQL.

## Estilo de código
- Python sencillo y claro. Sin trucos de listo, sin one-liners crípticos.
- Todos los comentarios en español.
- Cada bloque de código debe ser comprensible para alguien sin nivel alto de desarrollo.
- Preferir claridad sobre brevedad siempre.

## Cómo trabajar conmigo
- Antes de escribir código, explicar el QUÉ y el POR QUÉ de lo que se va a hacer.
- Avanzar sección por sección, no implementar todo de golpe.
- Esperar mi confirmación ("OK", "adelante", "sí") antes de aplicar cambios grandes.
- Si hay varias formas de hacer algo, explicar las opciones brevemente antes de elegir.
- No asumir que entiendo algo: si hay un concepto importante detrás de una decisión, mencionarlo.

## Restricciones
- No generar código que no se haya explicado antes.
- No refactorizar ni reorganizar archivos sin pedirlo explícitamente.
- No instalar dependencias sin confirmación previa.
